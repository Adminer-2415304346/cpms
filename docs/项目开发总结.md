## 二、数据库设计

### 2.1 数据库信息

| 项目 | 内容 |
|------|------|
| 数据库名 | `cpms` |
| 字符集 | utf8mb4 |
| 用户 | `cpms_admin@localhost` |
| 建库脚本 | `docs/schema.sql` |

### 2.2 数据表（12 张）

| 序号 | 表名 | 说明 | 核心字段 |
|------|------|------|----------|
| 1 | `user` | 用户表 | user_id, username, password, role(admin/staff/owner), real_name, phone, email, status, created_at |
| 2 | `owner` | 业主表 | owner_id, user_id(FK), name, id_card, phone, move_in_date |
| 3 | `building` | 楼栋表 | building_id, building_name, floors, description |
| 4 | `room` | 房间表 | room_id, building_id(FK), unit_name, room_number, area, room_type, status |
| 5 | `owner_room` | 业主-房间关联 | id, owner_id(FK), room_id(FK), relation_type, start_date, end_date |
| 6 | `bill` | 账单表 | bill_id, room_id(FK), fee_type, amount, period, due_date, status, created_at |
| 7 | `payment` | 缴费记录表 | payment_id, bill_id(FK), amount, pay_method, pay_time, staff_id(FK) |
| 8 | `repair` | 报修表 | repair_id, owner_id(FK), room_id(FK), description, urgency, status, staff_id(FK), created_at, completed_at |
| 9 | `complaint` | 投诉建议表 | complaint_id, owner_id(FK), type, content, is_anonymous, status, reply, created_at, replied_at |
| 10 | `announcement` | 公告表 | announcement_id, title, content, publisher_id(FK), publish_time |
| 11 | `parking_spot` | 停车位表 | spot_id, spot_code, location, type, status |
| 12 | `vehicle` | 车辆表 | vehicle_id, owner_id(FK), spot_id(FK), plate_number, vehicle_type, color |

### 2.3 数据库对象

| 类型 | 名称 | 说明 |
|------|------|------|
| **视图** | `v_overdue_bills` | 欠费催缴一览（关联 5 表，展示欠费业主、房号、金额、逾期天数） |
| **触发器** | `tr_payment_after_insert` | 插入缴费记录后自动更新账单状态为"已缴" |
| **索引** | 14 个 | 覆盖外键、登录查询、状态筛选等高频查询 |

### 2.4 设计文档

| 文件 | 内容 |
|------|------|
| `docs/er-diagram.md` | ER 图 Mermaid 代码（完整 12 实体 + 关系） |
| `docs/schema.sql` | 完整建库建表脚本（含索引/视图/触发器/示例数据） |

## 三、后端 API 设计

### 3.1 项目结构

```
cpms-backend/
├── pom.xml
├── src/main/java/com/cpms/
│   ├── CpmsApplication.java
│   ├── config/
│   │   ├── CorsConfig.java              # CORS 跨域（允许所有来源）
│   │   └── GlobalExceptionHandler.java  # 全局异常处理（统一返回 ApiResult）
│   ├── entity/          (12 个 JPA 实体)
│   ├── repository/      (12 个 JPA 接口)
│   ├── service/         (10 个业务类)
│   ├── controller/      (10 个控制器)
│   └── dto/
│       ├── ApiResult.java       # 统一响应 {code, message, data}
│       ├── LoginRequest.java    # 登录请求
│       └── LoginResponse.java   # 登录响应
└── src/main/resources/
    └── application.properties
```

### 3.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

### 3.3 API 端点清单

#### 认证模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 用户登录（返回 userId, username, role, realName） |

#### 用户管理（管理员）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/users` | 用户列表（可选 `?role=staff` 筛选） |
| GET | `/api/users/{id}` | 用户详情 |
| POST | `/api/users` | 创建用户 |
| PUT | `/api/users/{id}` | 编辑用户 |
| DELETE | `/api/users/{id}` | 删除用户 |

#### 楼栋管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/buildings` | 楼栋列表 |
| GET | `/api/buildings/{id}` | 楼栋详情 |
| POST | `/api/buildings` | 新建楼栋 |
| PUT | `/api/buildings/{id}` | 编辑楼栋 |
| DELETE | `/api/buildings/{id}` | 删除楼栋 |

#### 房间管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/rooms` | 房间列表（可选 `?buildingId=1` 筛选） |
| GET | `/api/rooms/{id}` | 房间详情 |
| POST | `/api/rooms` | 新建房间 |
| PUT | `/api/rooms/{id}` | 编辑房间 |
| DELETE | `/api/rooms/{id}` | 删除房间 |

#### 业主管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/owners` | 业主列表 |
| GET | `/api/owners/{id}` | 业主详情 |
| GET | `/api/owners/by-user/{userId}` | 根据用户ID查业主 |
| POST | `/api/owners` | 创建业主 |
| PUT | `/api/owners/{id}` | 编辑业主 |
| DELETE | `/api/owners/{id}` | 删除业主 |

#### 业主-房间绑定

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/owner-rooms/by-owner/{ownerId}` | 业主名下的房间 |
| GET | `/api/owner-rooms/by-room/{roomId}` | 房间绑定的业主 |
| POST | `/api/owner-rooms` | 绑定业主与房间 |
| DELETE | `/api/owner-rooms/{id}` | 解绑 |

#### 账单管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/bills` | 账单列表（可选 `?roomId=` / `?buildingId=` / `?unpaid=true`） |
| GET | `/api/bills/{id}` | 账单详情 |
| POST | `/api/bills` | 生成账单 |
| PUT | `/api/bills/{id}` | 编辑账单 |
| POST | `/api/bills/{id}/pay` | 缴费（会触发触发器更新账单状态） |
| DELETE | `/api/bills/{id}` | 删除账单 |

#### 报修管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/repairs` | 报修列表（可选 `?ownerId=` / `?status=` / `?staffId=`） |
| GET | `/api/repairs/{id}` | 报修详情 |
| POST | `/api/repairs` | 提交报修 |
| PUT | `/api/repairs/{id}/assign` | 派工（→ assigned） |
| PUT | `/api/repairs/{id}/start` | 开始维修（→ in_progress） |
| PUT | `/api/repairs/{id}/complete` | 维修完成（→ completed） |
| PUT | `/api/repairs/{id}/confirm` | 业主确认（→ confirmed） |
| DELETE | `/api/repairs/{id}` | 删除工单 |

**报修状态流转**: `pending` → `assigned` → `in_progress` → `completed` → `confirmed`

#### 投诉建议

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/complaints` | 投诉列表（可选 `?ownerId=` / `?status=`） |
| GET | `/api/complaints/{id}` | 投诉详情 |
| POST | `/api/complaints` | 提交投诉/建议 |
| PUT | `/api/complaints/{id}/reply` | 物业回复（→ resolved） |
| DELETE | `/api/complaints/{id}` | 删除 |

#### 公告管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/announcements` | 公告列表（按时间倒序） |
| GET | `/api/announcements/{id}` | 公告详情 |
| POST | `/api/announcements` | 发布公告 |
| DELETE | `/api/announcements/{id}` | 删除公告 |

#### 停车管理

**车位：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/parking/spots` | 车位列表（可选 `?available=true`） |
| GET | `/api/parking/spots/{id}` | 车位详情 |
| POST | `/api/parking/spots` | 新增车位 |
| PUT | `/api/parking/spots/{id}` | 编辑车位 |
| DELETE | `/api/parking/spots/{id}` | 删除车位 |

**车辆：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/parking/vehicles` | 车辆列表（可选 `?ownerId=`） |
| GET | `/api/parking/vehicles/{id}` | 车辆详情 |
| POST | `/api/parking/vehicles` | 登记车辆（绑定车位） |
| PUT | `/api/parking/vehicles/{id}` | 编辑车辆 |
| DELETE | `/api/parking/vehicles/{id}` | 删除车辆 |

## 四、前端页面（已完成）

### 4.1 技术栈

| 项 | 内容 |
|----|------|
| 框架 | Vue 3 + Vite + JavaScript |
| UI 库 | Ant Design Vue 4.x |
| 状态管理 | Pinia（用户持久化） |
| HTTP 客户端 | Axios（统一拦截器） |
| 样式 | 紫蓝主题全局 CSS + 玻璃态 + 粒子动画 |

### 4.2 页面清单

| 模块 | 页面 | 路由 | 功能 |
|------|------|------|------|
| 登录 | 登录页 | `/login` | 深蓝渐变背景 + 玻璃态卡片 + 紫金学院标识（龙形 SVG） |
| 首页 | 仪表盘 | `/dashboard` | 统计卡片 / 快速操作 / 最近工单 |
| 用户管理 | 用户列表 / 编辑 | `/users` | CRUD / 角色筛选（admin/staff/owner） |
| 楼栋管理 | 楼栋列表 / 编辑 | `/buildings` | CRUD / 楼层数设定 |
| 房间管理 | 房间列表 / 编辑 | `/rooms` | CRUD / 按楼栋筛选 / 绑定业主 |
| 业主管理 | 业主列表 / 编辑 | `/owners` | CRUD / 关联用户账户 |
| 账单管理 | 账单列表 / 生成 / 缴费 | `/bills` | CRUD / 费用类型 / 缴费触发状态更新 |
| 报修管理 | 工单列表 / 状态流转 | `/repairs` | 5 步状态流转（pending→assigned→in_progress→completed→confirmed） |
| 投诉管理 | 投诉列表 / 回复 | `/complaints` | 提交投诉/建议 / 物业回复 |
| 公告管理 | 公告列表 / 发布 | `/announcements` | 发布公告 / 按时间倒序 |
| 停车管理 | 车位 / 车辆 | `/parking` | 车位 CRUD / 车辆登记与绑定 |

### 4.3 项目结构

```
cpms-frontend/
├── src/
│   ├── api/           # Axios 封装 + 各模块 API 调用
```
