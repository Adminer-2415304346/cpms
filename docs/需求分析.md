# 社区物业管理系统 - 需求分析

## 1. 系统概述

### 1.1 项目背景

随着城市化进程加快，住宅社区规模不断扩大，传统的纸质化物业管理方式效率低下，信息传递不及时，已无法满足现代社区管理需求。社区物业管理系统（Community Property Management System，简称CPMS）旨在通过信息化手段，实现社区物业管理的数字化、规范化，提高物业管理效率和服务质量。

### 1.2 系统目标

- 实现社区房产、业主信息的集中管理与维护
- 提供物业费用的在线生成、收缴与查询功能
- 建立报修、投诉的线上处理流程，提升服务响应速度
- 实现公告通知的及时发布，方便业主获取社区动态
- 规范化停车位管理，避免车位使用纠纷
- 为不同角色提供差异化的功能权限

### 1.3 可行性分析

| 维度 | 分析 |
|------|------|
| **技术可行性** | 采用成熟的 Web 技术栈（Vue3 + Spring Boot + MySQL），开发人员具备相关技能，技术风险低 |
| **经济可行性** | 全部使用开源技术，无需购买商业授权，开发与部署成本低 |
| **操作可行性** | B/S 架构，用户通过浏览器访问，操作界面友好，无需安装客户端 |

## 2. 用户角色分析

系统包含三个用户角色：

### 2.1 角色定义

| 角色 | 账号前缀 | 说明 |
|------|----------|------|
| **系统管理员** (admin) | admin_ | 系统最高权限者，负责系统维护和用户管理 |
| **物业人员** (staff) | staff_ | 物业公司员工，负责日常运营操作 |
| **业主** (owner) | owner_ | 社区住户，通过系统查看信息、提交申请 |

### 3.2 各模块详细描述

#### 3.2.1 用户与权限管理

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理系统的用户账号、角色分配和登录认证 |
| **输入** | 用户名、密码、角色类型、个人信息 |
| **处理** | 登录时验证用户名密码，根据角色加载对应权限菜单；管理员可创建/禁用/删除用户账号 |
| **输出** | 登录成功进入对应角色的首页；登录失败提示错误信息 |
| **约束** | 密码需加密存储，首次登录建议修改密码 |

#### 3.2.2 房产信息管理

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理社区内的楼栋、单元、房间层级结构信息 |
| **输入** | 楼栋编号/名称、单元号、房间号、建筑面积、户型等 |
| **处理** | 支持三级层级（楼栋→单元→房间）的增删改查操作 |
| **输出** | 房产信息列表、详情、按条件查询结果 |
| **约束** | 有业主入住的房间不可直接删除 |

#### 3.2.3 业主信息管理

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理业主个人档案及与房产的关联关系 |
| **输入** | 姓名、身份证号、联系电话、入住时间、关联房产 |
| **处理** | 登记/编辑业主信息，建立业主与房间的归属关系 |
| **输出** | 业主信息列表、业主详情（含名下房产） |
| **约束** | 身份证号在系统中唯一 |

#### 3.2.4 物业费用管理

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理物业相关费用的生成、收缴和查询 |
| **输入** | 费用类型、金额、所属业主/房产、计费周期 |
| **处理** | 物业人员生成账单→业主查看→缴费→物业确认收款 |
| **输出** | 账单列表、缴费记录、欠费统计 |
| **约束** | 同一周期同一费用类型不可重复生成账单 |

#### 3.2.5 报修管理

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理业主报修申请及物业处理流程 |
| **输入** | 报修描述、位置、紧急程度、联系方式 |
| **处理** | 业主提交→物业人员接单/派工→维修中→完成→业主确认 |
| **输出** | 报修工单列表、状态流转记录 |
| **约束** | 已完成工单不可再次编辑 |

#### 3.2.6 投诉建议

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理业主投诉与建议的提交与处理 |
| **输入** | 类型（投诉/建议）、内容描述、联系方式 |
| **处理** | 业主提交→物业人员查看并处理→反馈处理结果 |
| **输出** | 投诉建议列表、处理详情 |
| **约束** | 支持匿名提交 |

#### 3.2.7 公告通知

| 项目 | 内容 |
|------|------|
| **功能描述** | 物业发布社区公告，业主查看公告 |
| **输入** | 标题、内容、发布时间、发布人 |
| **处理** | 物业人员新建/删除公告；所有用户可查看公告列表和详情 |
| **输出** | 公告列表（按时间倒序）、公告详情 |
| **约束** | 已发布的公告仅发布者和管理员可删除 |

#### 3.2.8 停车位管理

| 项目 | 内容 |
|------|------|
| **功能描述** | 管理社区停车位及车辆信息 |
| **输入** | 车位编号、位置、类型、车牌号、车主信息 |
| **处理** | 车位信息登记、车辆登记、车位与车辆绑定 |
| **输出** | 车位列表、车辆列表、使用记录 |
| **约束** | 一个车位同时只能绑定一辆车 |

## 4. 非功能需求

| 类别 | 需求 |
|------|------|
| **性能** | 常规页面响应时间 ≤ 2秒，支持 50 用户并发访问 |
| **安全性** | 密码加密存储；SQL 注入防护；权限访问控制；前端输入校验 + 后端二次校验 |
| **可用性** | 界面简洁直观，操作流程符合物业管理实际场景 |
| **可维护性** | 前后端分离架构，代码结构清晰，便于后续维护和功能扩展 |
| **兼容性** | 支持主流浏览器（Chrome、Edge、Firefox）最新版本 |
